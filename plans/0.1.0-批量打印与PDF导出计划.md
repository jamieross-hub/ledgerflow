# LedgerFlow v0.5.3 批量打印与 PDF 导出（A4）

> 创建日期：2026-03-14  
> 状态：已建立（待实施）  
> 目标：支持在「交易列表批量选择」场景下，生成 A4 规格的打印内容，并提供可用的“导出 PDF / 打印”能力（样式可控、分页稳定）。

---

## 1. 背景

- 交易列表已支持批量选择，并有「🖨️ 打印 A4」入口（现有实现需要确认覆盖范围与样式质量）。
- 需求补充：希望用户能把批量选择的账单生成**可打印的 A4 内容**，并支持**导出 PDF**（或通过打印对话框保存为 PDF），且渲染样式需要“像正式报表”，而不是简单 HTML。

## 2. 目标与原则

### 2.1 目标

1. 批量选择交易 → 一键生成 A4 打印版（分页稳定、样式完整）。
2. 在不引入大依赖/不做大重构的前提下，提供“导出 PDF”能力（最小可行：复用浏览器打印保存为 PDF）。

### 2.2 原则

- 最小侵入：优先复用现有批量打印逻辑（`handleBulkPrintA4` 等）。
- 样式可控：打印模板必须包含 `@page size: A4`、边距、字体、表格/条目布局与分页策略。
- 可验收：至少提供构建通过 + 明确的手动验收步骤；如可行再补最小自动化。

## 5. 风险与规避

- 风险：不同浏览器对打印 CSS 支持差异导致分页不一致。
  - 规避：优先兼容 Chrome；分页策略采用保守写法（`break-inside: avoid` 等）。
- 风险：PDF 真导出引入依赖（如 pdfkit/jspdf/html2canvas）体积大、兼容差。
  - 规避：P0 先走“打印保存为 PDF”，P1 再评估真导出。
