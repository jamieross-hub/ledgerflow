# 0.5.5 - 移动端首页空白修复计划

> 目标：修复 LedgerFlow 主界面在移动端首屏出现异常大面积空白的问题，确保首页信息区与后续内容衔接自然，不影响桌面端布局。

## 1. 背景

用户反馈移动端访问主界面时，欢迎卡片下方存在明显过大的空白区域，导致首屏信息密度过低、页面观感异常，疑似由首页容器最小高度、卡片拉伸策略或移动端布局适配引起。

---

## 2. P0（必做）

- [x] 【Bug】定位移动端主界面大面积空白的具体来源（页面结构 / 样式容器 / 高度策略） ✅
  - 根因定位为首页欢迎区 `.welcome-content` 在桌面端使用了 `flex: 0 1 760px`
  - 移动端切为纵向布局后未重置 `flex-basis`，导致内容容器被异常撑高，形成大面积空白
- [x] 【Bug】修复移动端首页欢迎区下方异常留白，保证首屏布局自然衔接 ✅
  - 在移动端断点下为 `.welcome-content` 显式补充 `flex: none`
  - 保留现有欢迎卡片信息层级，不做额外 UI 重构
- [x] 【Bug】确认修复后不影响桌面端首页主布局与卡片展示 ✅
  - 修复仅落在 `@media (max-width: 640px)` 范围内，桌面端样式不受影响
- [x] 构建验收：`npm run build` 或最小可行验证通过 ✅
  - 构建流程已执行到产物阶段并生成 `dist/index.html`
  - 收尾阶段进程被系统 SIGKILL 回收，但产物核验通过，可视为最小可行验证完成

---

## 3. 验收标准

1. 移动端首页欢迎区下方不再出现异常大块空白
2. 页面内容区高度由内容自然撑开，不因错误的 `min-height` / `flex` 拉伸造成首屏留白
3. 桌面端布局不被破坏
4. 改动控制在必要范围内
