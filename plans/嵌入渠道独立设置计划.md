# 嵌入渠道独立设置计划

> 创建日期：2026-03-14  
> 状态：已建立（待实施）  
> 目标：支持“嵌入（Embedding）模型/渠道”独立于当前聊天/推理渠道配置，避免嵌入能力被当前渠道商 baseUrl 绑定。

---

## 1. 背景与现状

- 现状：当前嵌入模型（用于记忆召回/语义检索等）的调用地址与配置，跟随“当前渠道商（聊天/推理渠道）”的 baseUrl/Key/模型来决定。
- 问题：
  - 当聊天渠道不支持 embedding，或 embedding 成本/质量不合适时，会影响记忆/检索稳定性。
  - 用户想使用 A 渠道聊天，但用 B 渠道做 embedding（例如更便宜/更稳定/更高质量）。

## 2. 目标与原则

- 目标：提供“嵌入渠道”独立配置：baseUrl / apiKey / embeddingModel（以及必要的维度/批量等参数）。
- 原则：
  - 最小侵入：不重构现有渠道体系，只在 embedding 相关调用处增加“优先读嵌入渠道配置”的选择逻辑。
  - 默认不破坏：未配置嵌入渠道时，行为保持与现状一致（仍跟随当前渠道商）。
  - 可验证：提供可运行的连接测试/最小校验，避免静默失败。

---

## 3. P0：嵌入渠道配置落地（必做）

- [x] 设计配置结构：新增 `embedding` 专用配置（baseUrl/apiKey/model 等） ✅
- [ ] 配置入口：在设置页提供“嵌入渠道”单独配置与启用开关
- [ ] 调用改造：embedding 请求优先使用嵌入渠道配置；未配置则回落到当前渠道配置
- [ ] 连接测试：提供“测试嵌入配置”按钮（最小调用 + 明确报错提示）
- [ ] 回归：记忆召回/语义检索链路在未配置与已配置两种情况下都可用

### P0 验收标准

- 未配置嵌入渠道：产品行为与当前一致（不报错、不改变结果路径）。
- 已配置嵌入渠道：embedding 请求明确走嵌入渠道；聊天仍走原渠道。
- 失败可见：embedding 配置错误时，有明确可理解的错误提示。

---

## 4. P1：体验与稳定性增强（可选）

- [ ] 缓存与降级：embedding 失败时回落策略更清晰（例如提示并禁用记忆注入，而非全链路失败）
- [ ] 多模型选择：支持常见 embedding 模型名提示/校验（避免手输错误）
- [ ] 隐私提示：提示嵌入渠道可能上传文本用于向量化（与聊天渠道可能不同）

---

## 5. 风险与规避

- 风险：引入第二套 Key/URL 后，用户配置更复杂。
  - 规避：默认隐藏为“高级选项”；提供测试按钮与示例。
- 风险：不同渠道 embedding 维度不同导致向量库不一致。
  - 规避：记录 embedding 维度/模型信息；必要时提示“切换 embedding 模型需重建向量”。
