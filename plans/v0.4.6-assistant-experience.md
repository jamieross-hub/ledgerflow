# LedgerFlow v0.4.6 AI 助手与记账体验优化计划

> 创建日期：2026-03-06  
> 状态：下一阶段候选主线（聚焦 **AI 助手连续问答体验、流式输出、首页去冗余、快捷操作增强、记账页体验收口**）。

---

## 1. 当前版本目标

`v0.4.6` 聚焦把 LedgerFlow 从“能用的 AI 记账工具”继续打磨成“更顺手、更少打断、更少重复内容”的产品。

这版重点不是再堆新模块，而是围绕主流程收口：

1. AI 助手回答后继续问下一轮更自然
2. AI 输出过程更像实时生成，而不是整段突然出现
3. 首页减少冗余与重复内容
4. 键盘操作更完整
5. “记一笔”页面更顺手
6. AI 记账保存后不再把长 JSON 残留在前端界面

核心原则：

- 优先减少打断感与重复感
- 优先优化用户最常见的主路径
- 保持功能解释清楚，但不要让界面像调试台
- 能在前端收口的问题先前端收口

## 2. P0：AI 助手连续问答与输出体验

### 2.1 AI 回答下方增加联想词 / 追问词 ✅
目标：

在 AI 助手每次回答下方增加联想词或追问词按钮，用户点击后可直接进入下一轮问答。

重点处理：

- 联想词显示在助手回答下方，不影响正文阅读
- 点击后自动填入并直接发起下一轮问答
- 联想词数量控制在 3~5 个，避免过多
- 联想词内容应与当前回答语义相关，而不是固定模板
- 移动端也要保证点击区域清晰

验收标准：

- 助手回答下方可看到联想词
- 点击后能自动触发下一轮提问
- 不出现和回答完全无关的词

### 2.2 AI 助手支持流式响应输出 ✅
目标：

让 AI 助手回答支持流式输出，减少“整段突然出现”的割裂感。

重点处理：

- 优先支持回答正文流式输出
- 流式中断、失败、重试状态要可解释
- 保持与现有消息记录逻辑兼容
- 避免因为流式改造导致重复消息或残缺消息

验收标准：

- AI 回答可逐步输出
- 中断与失败时状态清楚
- 不破坏当前历史消息逻辑

### 2.3 AI 记账保存后清理前端长 JSON 残留
目标：

解决 AI 记账保存账单后，前端界面中仍残留大段 JSON 的问题，避免像调试页面。

重点处理：

- 保存成功后清理调试态 / 原始 JSON 残留
- 保留用户真正需要看到的结果摘要
- 如需保留调试信息，应折叠到次级区域而非常驻
- 区分“保存前预览”与“保存后结果反馈”两个阶段

验收标准：

- 保存后前端不再滞留大段 JSON
- 用户还能看懂保存结果
- 不影响排障能力

## 3. P1：首页与信息架构去冗余

### 3.1 移除首页冗余内容，避免多处重复表达同一功能
目标：

对首页做一轮去冗余整理，避免同一类能力、说明或入口在多个地方重复出现。

重点处理：

- 盘点首页与帮助页、助手页、交易页之间的重复内容
- 首页只保留真正高频且值得首屏展示的信息
- 已迁移到帮助页 / 助手页的内容，不再在首页重复讲一遍
- 优先保留“总览、行动、入口”，弱化重复说明

验收标准：

- 首页信息密度更合理
- 重复内容明显减少
- 首页与帮助页职责更清晰

---

## 4. P1：快捷操作与主流程效率

### 4.1 增加更多键盘快捷方式
目标：

为核心高频操作补充更多键盘快捷方式，降低切页和鼠标依赖。

重点处理：

- 盘点已有快捷键并补全缺口
- 候选包括：进入 AI 助手、保存、切换模式、聚焦搜索、快速回首页等
- 避免与浏览器原生快捷键强冲突
- 在帮助页统一展示

验收标准：

- 高频路径拥有更完整快捷键
- 不与常用浏览器快捷键明显冲突
- 帮助页能查到快捷键说明

### 4.2 优化“记一笔”页面 UI 与体验
目标：

对“记一笔”页面做一轮结构、交互和视觉整理，让录入路径更顺手。

重点处理：

- 优化字段布局与主次顺序
- 减少视线来回跳转
- 优化按钮、金额输入、分类选择、账户选择等关键路径
- 提高桌面端与移动端录入效率

验收标准：

- “记一笔”页面更顺手
- 输入路径更短、更少打断
- 页面视觉与现有成熟页面更统一

## 6. 执行原则

1. 每完成一项都更新计划状态
2. 每一轮修改后都执行 `npm run build`
3. AI 助手相关改造优先保证消息流稳定
4. 首页去冗余时避免误删真正高频入口
5. 文案、动效、结构三者保持同一产品语言
