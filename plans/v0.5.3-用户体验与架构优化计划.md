# LedgerFlow v0.5.3 用户体验与架构优化计划

> 创建日期：2026-04-06
> 状态：进行中
> 目标：从忠实用户视角，优化体验痛点、修复架构问题、提升可维护性

---

## 1. 审查发现概述

作为忠实用户，经过全面代码审查，发现以下主要问题：

### 1.1 用户体验问题
- **核心页面性能**：DashboardPage 2027 行、TransactionsPage 3390 行、AssistantPage 3327 行，单文件过大影响加载和渲染
- **样式系统**：global.css 12724 行，职责边界不清，维护困难
- **移动端体验**：已有专门修复（0.5.5），但部分页面仍有优化空间
- **功能入口**：部分功能入口深度过深，用户发现成本高

### 1.2 架构与可维护性问题
- **代码组织**：超大页面组件职责混杂，难以测试和迭代
- **状态管理**：部分业务逻辑与 UI 耦合过深
- **类型安全**：整体良好，但部分边界情况缺少类型保护
- **测试覆盖**：核心功能有测试，但关键用户流程（如移动端布局、PDF 导出端到端）缺少回归保护

### 1.3 已有进展（从 0.5.6 审查计划继承）
- ✅ PDF 导出逻辑已拆分到 `exportTransactionsPdf.ts`
- ✅ `pdf-lib` 改为按需加载
- ✅ 首页分类结构区错位问题已修复
- ✅ vite 分包策略已补充维护注释

## 6. 验收标准

1. **架构可维护性**：核心页面文件行数减少 50% 以上
2. **功能完整性**：所有现有功能正常工作，无回归
3. **性能基线**：首屏加载时间不退化，交互响应时间 < 100ms
4. **测试覆盖**：关键路径有测试保护
5. **用户体验**：常用功能步数减少，发现成本降低
