# 计划目录说明

LedgerFlow 计划目录已调整为“单一主线计划 + 完成归档 + 远期储备”的维护方式。

## 当前目录约定

- `plans/当前主线计划.md`：唯一主维护入口，统一记录当前未完成事项、页面规范流程、发版流程
- `plans/已完成汇总.md`：已完成事项归档，按任务维度持续追加
- `plans/v0.6.0-user-system-collaboration.md`：远期储备方向，当前不纳入主线执行
- `plans/archive/`：历史归档与兼容入口，仅用于追溯

## 优先阅读顺序

1. `plans/当前主线计划.md`
2. `plans/已完成汇总.md`
3. `plans/v0.6.0-user-system-collaboration.md`（仅在评估远期能力时查看）
4. `plans/archive/`（仅在追溯历史背景时查看）

## 维护原则

1. `plans/` 根目录不再同时维护多个并行主计划文件
2. 当前未完成事项统一收敛到 `plans/当前主线计划.md`
3. 某项完成后，先更新主线状态，再追加到 `plans/已完成汇总.md`
4. 已完成或失去主线价值的旧计划文件，不再保留在 `plans/` 根目录
5. 远期方向可单独保留，但不得与当前主线混写

## 命名与落点规则

- 当前主线：固定使用 `plans/当前主线计划.md`
- 已完成归档：固定写入 `plans/已完成汇总.md`
- 远期储备：按版本或主题单独命名，例如 `plans/v0.6.0-user-system-collaboration.md`
- 历史兼容文件：放在 `plans/archive/`

## 使用方式

- 做当前需求：更新 `plans/当前主线计划.md`
- 记录完成结果：更新 `plans/已完成汇总.md`
- 评估未来大方向：查看对应远期计划文件
- 追溯旧方案：进入 `plans/archive/`
