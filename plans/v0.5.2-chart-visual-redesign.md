# LedgerFlow v0.5.2 图表视觉重构计划

> 创建日期：2026-03-07  
> 状态：已建立，待切主线（聚焦 **趋势图重构、支出分类构成图重构、图表视觉语言统一**）。

---

## 1. 当前版本目标

`v0.5.2` 聚焦图表本身的表达方式重构，不只是“换皮肤”，而是把目前偏功能化、组件拼装感较强的图表，统一成更克制、更清晰、更有产品感的视觉系统。

首批重点落两类核心图表：

1. **趋势图改为极简单轴柱状图**
2. **支出分类构成图改为环形图（Donut）+ 下方联动列表**

核心原则：

- 图表优先服务“快速理解”，不是展示图表控件能力
- 视觉层级应简洁，减少多余坐标、网格线、标签噪音
- 重点信息只高亮一个：峰值、主分类、核心金额
- 图表与下方列表必须形成联动，而不是各讲各的
- 允许更像产品插画式图表，但不能牺牲可读性

## 2. P0：趋势图（柱状图）重构

### 2.1 月度内每日支出趋势图改为单轴柱状图
目标：

将趋势图改为用于展示“月度内每日支出时间分布”的单轴柱状图。

视觉描述：

- 类型：单轴柱状图
- 用途：展示月度内每日支出的时间分布
- 横轴：等距分布的日期刻度点，用细灰色竖线作为基线
- 纵轴：无明确刻度，通过柱子高度直观体现数值大小
- 高亮：
  - 其中一根柱子使用主色（蓝色）填充
  - 高度显著高于其他柱子，代表单日支出峰值
  - 其余柱子为浅灰色细条，高度极低或为零
- 交互提示：顶部有一个圆角矩形悬浮提示条，用于显示选中日期详情
- 风格：极简，无网格线，无坐标轴标签，仅保留时间与高度对比关系

重点处理：

- 替换当前偏折线 / 混合式趋势表达
- 明确“默认高亮峰值”与“悬浮切换选中日”的交互逻辑
- 小屏下仍要保持稀疏、稳定，不变成一排难点的细线
- Tooltip 文案要简洁，不要信息过载

验收标准：

- 用户能快速看出哪一天支出最高
- 没有冗余网格、坐标和装饰噪音
- 图表即使不看数字，也能读出趋势分布

## 3. P0：支出分类构成图（环形图）重构

### 3.1 分类构成图改为 Donut Chart + 联动列表
目标：

将支出分类构成改为更聚焦占比关系的环形图，并配合下方列表形成视觉联动。

视觉描述：

- 类型：环形图（Donut Chart）
- 用途：展示不同支出分类占比
- 中心：白色圆形区域，用作总支出金额占位
- 外环：
  - 由不同色块扇形组成
  - 主色蓝色扇形占据绝大部分面积（示例约 95%）
  - 浅灰色小扇形占比极小（示例约 5%）
- 标注：
  - 从每个扇形边缘延伸出细线
  - 指向对应分类图标（如钱袋、碗）和占比占位
- 下方列表：
  - 左侧：分类图标
  - 中间：进度条（长度与占比对应，主色填充）
  - 右侧：金额占位符与笔数占位符
- 联动：
  - 进度条长度与环形图扇形占比完全对应
  - 形成直觉一致的视觉映射

重点处理：

- 替换当前饼图 / 条形占比的割裂表达
- Donut 中心区域要承担总额展示，不空着浪费
- 图例不再只是普通 legend，而是升级为可读列表
- 颜色策略要统一，主分类重点突出，其余分类克制分层

验收标准：

- 用户能快速读出主分类占比
- 环形图与列表有明显联动关系
- 信息层级比现状更清楚，不像图表 demo

## 4. P1：图表视觉语言统一

### 4.1 统一图表色彩、圆角、提示条与占位风格
目标：

让 Dashboard / Transactions / Smart Budget 等页面中的图表风格趋于统一。

重点处理：

- 主色、次色、灰阶使用统一规范
- Tooltip / 悬浮条统一圆角、阴影、字级
- 图表的空数据态、低数据态样式统一
- 柱条、环形、进度条的圆角表达统一

### 4.2 小屏图表优先保证识别，不强塞信息
目标：

在移动端下，优先保留可读性，不强行堆满标签和注释。
