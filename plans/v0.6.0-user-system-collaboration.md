# LedgerFlow v0.6.0 用户系统与协作能力计划

> 创建日期：2026-03-07  
> 状态：已建立，暂不着急推进（聚焦 **用户功能、身份体系、协作与共享基础能力**）。

---

## 1. 当前版本目标

这个版本先**登记，不急着做**。

它用于承接未来更重的一类能力：

1. 用户身份体系
2. 多设备 / 多端同步基础
3. 协作与共享账本
4. 用户级设置、权限和数据隔离

之所以单独立项，是为了避免这些需求被零散塞进当前版本，导致本地产品还没打稳，就提前把系统复杂度拉爆。

核心原则：

- 用户功能是大工程，不和当前单用户主线混做
- 先把本地数据结构、分享能力、远程数据库边界想清楚，再谈用户系统
- 未来若做协作，权限、隔离与审计必须先于花哨入口

---

## 2. 后续可扩展方向

### 2.1 用户账号与登录
- 邮箱 / 手机号 / 第三方登录
- 基础身份识别与设备绑定

### 2.2 用户级数据隔离
- 每个用户独立账本
- 用户级偏好、AI 设置、记忆与附件隔离

### 2.3 多人共享账本 / 家庭账本
- 邀请成员
- 权限分级（查看 / 编辑 / 管理）
- 操作留痕

### 2.4 分享升级为协作
- 分享账单从“只读导出”升级到“受控共享”
- 讨论、备注、审阅状态等扩展能力

---

## 3. 执行原则

1. 先登记，暂不切主线
2. 等本地版、远程连接、分享能力更成熟后再评估优先级
3. 若后续切主线，需先补一版架构设计文档
