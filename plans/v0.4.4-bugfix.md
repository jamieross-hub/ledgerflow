# LedgerFlow v0.4.4 Bug 修复计划

> 创建日期：2026-03-06  
> 状态：当前阶段以 **bug 修复、细节校正、页面专业度补强** 为第一优先级。

---

## 1. 当前版本目标

`v0.4.4` 不再以大功能扩展为重点，而是集中处理上一版发布后暴露出来的：

1. 交互 bug
2. 图表显示异常
3. 移动端挤压与布局问题
4. 低专业感页面的结构整理
5. 首页与帮助入口的信息架构优化

核心原则：

- 优先修“用户一眼就能看到不对劲”的问题
- 优先修“已发布后最影响信任感”的问题
- 避免在本版本再引入大而重的功能分叉
- 每一项都应可单独验证、单独提交

---

## 2. P0：发布后直接影响体验的问题

### 2.1 账单详情测试 / WebDAV 上传失败（HTTP 400） ✅
目标：

修复账单详情附件上传链路中出现的 WebDAV 400 问题，确保测试链路与真实上传链路一致。

重点排查：

- 上传路径拼接是否异常
- 文件名 / 编码 / content-type 是否导致 WebDAV 拒绝
- 请求 method / body 是否符合当前 WebDAV 服务要求
- 测试桩与真实实现是否存在行为偏差
- 错误提示是否足够明确，避免只看到 400 却无法定位原因

验收标准：

- 能稳定复现并定位 400 原因
- 修复后上传成功或失败提示可解释
- 相关测试更新通过
- `npm run build` 通过

### 2.2 收支趋势（决策视图）柱高异常 ✅
目标：

修复首页 / Dashboard 中收支趋势柱子看起来高度一致的问题，确保图形真实反映数据差异。

重点排查：

- 高度归一化逻辑是否错误
- 最大值 / 最小值计算是否被固定值覆盖
- 正负值、极小值、空值处理是否导致统一高度
- 样式层是否把柱高截断为同一值

验收标准：

- 不同月份数据能体现明显高低差
- 极端值与小值都能合理显示
- 不出现溢出或统一高度假象

### 2.3 分类占比饼图（当前列表）显示异常 ✅
目标：

修复交易记录页分类占比饼图只显示最大面积、其他面积不显示的问题。

重点排查：

- 渐变分段计算是否错误
- 百分比归一化是否被单项吞掉
- 颜色 stop 范围是否重叠 / 覆盖
- 小比例分类是否被错误过滤

验收标准：

- 饼图能正确显示多个分类面积比例
- 图例与图形一致
- 空数据和极小占比时不崩、不假显示

## 3. P1：移动端与页面细节校正

### 3.1 时间轴模式 icon 与竖线未对齐 ✅
目标：

修复时间轴模式中 icon 与竖线视觉不对齐的问题，避免出现明显毛刺感。

重点处理：

- 节点圆点 / 图标垂直居中
- 竖线起点终点与节点位置统一
- 不同内容高度下的对齐稳定性
- 移动端与桌面端都要一致

验收标准：

- 时间轴节点与竖线视觉对齐
- 不因文案换行而错位
- 首项 / 末项显示稳定

### 3.2 移动端 AI 助手界面过挤 ✅
目标：

压缩移动端 AI 助手界面噪音，把主要空间还给对话内容与输入区。

补充约束（2026-03-06）：

- AI 助手页“你好，我是你的财务助手”卡片组后续需支持 PC 端宽屏左右对齐布局，窄屏自动收紧
- 语义召回信息不应强制常驻在 AI 助手主界面
- “最近一笔交易”不应固定显示在输入框上方，后续改为动态融入输入区

重点处理：

- “AI 记账助手”改到顶部标题区域
- 底部语义召回栏移除、折叠或显著缩小
- 顶部状态条、模式切换、说明区避免堆叠过高
- 移动端首屏优先保证消息区与输入区

验收标准：

- 移动端首屏不再被顶部/底部辅助条挤压
- 助手标题位置更自然
- 语义召回信息不再抢主界面空间

### 3.3 首页欢迎栏居中 + 主题色背景 ✅
目标：

调整“夜深了，注意休息，欢迎使用 LedgerFlow”这一栏的对齐与背景样式，使其更像正式欢迎区，而不是普通提示条。

重点处理：

- 文案水平居中
- 背景加入主题色，但不刺眼
- 保持暗色 / 浅色主题一致性
- 手机端避免挤压或断行难看

验收标准：

- 视觉中心明确
- 与首页整体风格一致
- 不再显得像临时提示条

### 3.4 关于 LedgerFlow 页面版本号未更新 ✅
目标：

修复关于页版本号未同步到 `v0.4.3 / v0.4.4` 相关状态的问题，避免发版后版本展示失真。

验收标准：

- 关于页展示的版本号与当前发布版本一致
- 不存在多个页面版本信息不一致的问题

## 4. P1：信息架构与功能可维护性优化

### 4.1 首页模块自定义：拖拽排序 + 开关卡片合并 ✅
目标：

把首页模块自定义中的拖拽排序与开关卡片整理成一个更统一的配置面板，减少割裂感。

重点处理：

- 拖拽排序与显隐开关整合到一个区域
- 列表层级更清楚
- 避免“排序一块、开关又一块”造成的重复认知成本

验收标准：

- 模块配置入口更集中
- 用户能同时完成排序与显隐调整
- 交互不比现在更复杂

### 4.2 新手三步移除，快捷方式迁移到帮助页面 ✅
目标：

从首页去掉“新手三步”，将快捷方式迁移到一个独立帮助页面，降低首页信息噪音。

重点处理：

- 首页去除新手三步区域
- 新建帮助页面承接快捷方式与使用引导
- 帮助页入口清晰，但不打断主流程
- 快捷方式、上手说明、常见问题可统一收纳

验收标准：

- 首页更干净
- 帮助入口明确
- 原有快捷能力没有丢失，只是迁移

### 4.4 WebDAV 同步备份版本化保留
目标：

让 WebDAV 同步备份从“单文件覆盖”升级为“带时间戳的版本化备份”，并允许用户设置最多保留的历史版本数量。

重点处理：

- 上传备份文件名自动加入时间戳，避免每次覆盖同一文件
- 支持配置“最多保留 N 个版本”
- 超出保留数量时，自动删除最旧版本
- 保持当前恢复路径清晰，避免用户分不清哪个是最新备份
- 兼顾中文路径、空格、URL 编码与代理模式兼容性

验收标准：

- 每次上传到 WebDAV 的备份文件名可区分时间版本
- 用户可配置保留版本数量
- 超出数量后旧备份会被正确清理
- 下载/恢复流程仍清晰可用，不因版本化而变复杂

## 5. P2：专业感补强

### 5.1 分类结构（Top5 + 其他）缺少图表表达
目标：

让分类结构模块不只停留在文字列表，而是增加与数据规模匹配的图形表达。

重点处理：

- 在 Top5 + 其他 结构中加入图表 / 条形可视化
- 保持文字信息可读，不做花哨但无效的装饰
- 与现有图表语言统一

验收标准：

- 分类结构模块不再只有文字
- 用户能快速看出占比差异
- 图表不过度喧宾夺主

### 5.2 账户和分类页面专业度不足
目标：

对账户页与分类页做一轮结构与样式整理，使其更像正式后台配置页，而不是功能堆叠页。

重点处理：

- 统一头部、说明、分组、表单、列表层级
- 减少随意感和临时 patch 感
- 优化空状态、卡片间距、主次按钮

验收标准：

- 页面结构更清楚
- 表单和列表层级更专业
- 与 Dashboard / Transactions / Smart Budget 的成熟度差距缩小

## 7. 执行原则

1. 每完成一项就更新计划状态
2. 每一轮修改必须 `npm run build`
3. 若涉及图表修复，优先补最小验证
4. 优先保证移动端与已发布页面的稳定性
5. 本版本不引入和当前 bug 无关的大功能扩张
