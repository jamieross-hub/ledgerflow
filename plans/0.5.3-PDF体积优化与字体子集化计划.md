# 0.5.3 - PDF 体积优化与字体子集化计划

> 目标：在保留中文 PDF 导出能力的前提下，继续降低字体资源对包体积、加载链路和缓存压力的影响。

## 1. 背景

当前 0.5.2 已完成：
- 中文 PDF 导出能力
- 中文字体按需加载，避免静态进入主入口

但仍存在：
- 字体文件本身体积较大
- 首次导出 PDF 时仍需加载完整字体资源
- PWA / 构建产物总体积受到影响

因此需要补一个更小范围的后续计划，聚焦“字体文件本身的体积控制”。

---

## 2. P0（必做）

- [ ] 现状盘点：记录当前字体文件大小、构建产物变化、PDF 导出链路依赖关系
- [ ] 评估字体瘦身方案：
  - [ ] 方案 A：替换为更小的中文字体文件
  - [ ] 方案 B：引入子集化字体（subset）
  - [ ] 方案 C：按常用字符集预裁剪字体
- [ ] 确定最小可落地方案，并说明不采用其他方案的原因
- [ ] 实施字体体积优化，确保不破坏现有中文 PDF 导出
- [ ] 构建验收：`npm run build`
- [ ] 手动验收：中文交易导出 PDF，确认中文可读、无乱码、无回退

---

## 3. P1（可选）

- [ ] 若方案成本可控，补充字体加载失败时的降级提示
- [ ] 若方案收益明显，记录优化前后包体积对比数据

---

## 4. 验收标准

1. 中文 PDF 导出仍然可用
2. 构建通过
3. 字体资源或总体构建体积较当前方案有明确下降
4. 不引入服务端依赖，不做大重构
