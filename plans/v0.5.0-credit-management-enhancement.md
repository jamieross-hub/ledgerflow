# LedgerFlow v0.5.0 AI 信贷管理增强计划

> 创建日期：2026-03-07  
> 状态：已建立，待切主线（聚焦 **AI 信贷管理能力深化、年化利率/APR 等关键指标补强、提示词与测算链路增强**）。

---

## 1. 当前版本目标

`v0.5.0` 不再停留在“识别花呗 / 分期 / 贷款信息并带去还款管理”，而是把 AI 信贷管家真正往“可分析、可测算、可决策辅助”的方向推进一步。

这一版重点不是再堆更多炫技入口，而是补齐信贷管理真正有用的判断维度：

1. **年化利率 / APR / 月利率等关键指标更完整**
2. **贷款、分期、信用账单的测算结果更可操作**
3. **AI 信贷管家提示词更专业、更稳定、更少胡猜**
4. **还款管理页对信贷信息的呈现与分析更像“管理工具”而不是普通表单**

核心原则：

- 不把信贷分析做成玄学建议，优先给可验证、可追溯的测算结果
- AI 输出要明确“已识别”“推算值”“待确认”三种状态
- 年化利率、总利息、月供、剩余成本等指标要可解释
- 信贷提示词优先减少误判、漏字段、乱补全
- 优先强化现有 repayment-management 与 AI credit 模式，不另起炉灶

---

## 2. P0：信贷关键指标补强

### 2.1 增加年化利率 / APR / 月利率的统一表达 ✅
目标：

让信贷管理页和 AI 信贷管家不只是显示一个 `annualRate` 字段，而是能明确区分和展示：

- 年化利率（APR / IRR 口径可区分）
- 名义年利率
- 月利率
- 日利率（如适用）

重点处理：

- 梳理现有 `annualRate` 字段语义，避免混用
- 对不同类型信贷建立统一展示口径
- 对无法直接确认口径的数据标注“推算值 / 平台口径待确认”
- 在还款管理页展示时加入解释文案，避免用户误读

验收标准：

- 用户能看懂当前利率是“平台给出的”还是“系统推算的”
- 利率显示口径一致，不再模糊
- AI 回答中能引用这些字段而不混淆

### 2.2 增加总利息 / 月供 / 剩余成本等测算字段 ✅
目标：

补齐用户真正关心的几个问题：

- 这笔贷款总共会多花多少钱？
- 每个月大概要还多少？
- 剩余成本还有多少？
- 提前还款大概能省多少？

重点处理：

- 增加总利息、预计月供、剩余总成本等衍生指标
- 对等额本息 / 等额本金 / 最低还款等常见方式给出基础测算
- 对资料不足场景输出“待补字段”而不是强算
- 为后续 AI 分析和排序提供结构化指标

验收标准：

- 还款页能展示更完整的测算信息
- AI 可以围绕这些指标给出更像样的分析
- 缺数据时不胡算

## 3. P0：AI 信贷管家提示词强化

### 3.1 强化信贷识别提示词
目标：

让 AI 在识别贷款 / 花呗 / 信用卡分期 / 消费贷时，更稳定地提取关键字段，并少说废话。

重点处理：

- 明确要求优先提取：
  - 平台 / 产品名
  - 当前应还金额
  - 总欠款 / 剩余待还
  - 还款日 / 扣款日
  - 期数 / 剩余期数
  - 每期金额
  - 利息 / 服务费 / 年化利率（如可识别）
- 明确区分：
  - 原文明确出现
  - 根据结构推测
  - 无法判断待补充
- 禁止在识别结果中凭空补全关键数字
- 优化 JSON 输出稳定性，降低格式漂移

验收标准：

- 同类账单截图下输出更稳定
- 漏字段减少，胡编字段也减少
- 结构化结果更适合直接流入还款管理页

### 3.2 强化信贷分析提示词
目标：

不只会“识别账单”，还要更会回答这些问题：

- 哪些负债最值得优先处理？
- 哪些分期实际成本偏高？
- 哪些项目更应该提前还款？
- 现金流压力主要来自哪里？

重点处理：

- 增加围绕 APR、剩余成本、还款日集中度的分析框架
- AI 输出要优先给结论，再给依据，再给行动建议
- 对不确定内容明确写“建议核对”
- 引导 AI 少做情绪化鼓励，多给排序和判断依据

验收标准：

- AI 信贷回答更像财务分析助手，而不是普通聊天机器人
- 输出结构更清楚、决策价值更高
- 提示词可复用到还款建议、风险排查、还款优先级分析

### 3.3 补齐识别结果的保存按钮与跳转按钮
目标：

当前 AI 信贷管家在识别出意图或结构化结果后，用户还缺少明确的下一步动作入口。需要补齐“保存 / 带去管理 / 跳转查看”这类按钮，让识别结果不再悬空。

重点处理：

- 对识别出的信贷项目增加显式操作区
- 至少支持两类动作：
  - 保存/带入还款管理
  - 跳转到对应管理页继续处理
- 当识别结果不完整时，按钮文案也要区分“去补充”与“去保存”
- 保持移动端操作清晰，不把按钮藏得太深

验收标准：

- 用户识别完后，能立刻知道下一步怎么做
- 不再出现“识别出来了但没有保存按钮 / 跳转按钮”的断链感
- 识别 → 管理 的路径明显变短

## 4. P1：还款管理页继续增强

### 4.1 增加提前还款 / 还款策略模拟
目标：

让用户能快速比较不同还款策略的影响，例如：

- 本月多还 1000 元会怎样
- 优先还高利率 vs 优先还小余额
- 提前结清某笔贷款大约可省多少

### 4.2 增加信贷风险标签与解释
目标：

把“高利率 / 还款日过于集中 / 压力过高 / 信息缺失”这些风险，用更直接的标签方式呈现给用户。

## 5. 建议执行顺序

### Sprint A（指标定义）
1. 统一利率字段与展示口径
2. 增加总利息 / 月供 / 剩余成本测算

### Sprint B（AI 提示词）
3. 强化信贷识别提示词
4. 强化信贷分析提示词

### Sprint C（管理增强）
5. 增加提前还款 / 还款策略模拟
6. 增加风险标签与解释

---

## 6. 执行原则

1. 先统一字段语义，再强化 AI 提示词
2. 先让测算结果可靠，再让建议更智能
3. 所有 AI 输出都要区分“识别值 / 推算值 / 待确认”
4. 每完成一项都更新计划状态并执行 `npm run build`
