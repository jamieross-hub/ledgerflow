# LedgerFlow v0.4.8 测试补强与主题渐变优化计划

> 创建日期：2026-03-07  
> 状态：已建立，待切主线（聚焦 **测试补强、主题模式渐变效果、移动端 AI 助手进一步收紧**）。

---

## 1. 当前版本目标

`v0.4.8` 计划在不打断当前主功能线的前提下，补三类“长期收益但不适合临时插队”的优化：

1. 增加更多关键测试文件，减少 UI / Assistant / WebDAV / 还款链路回归风险
2. 给主题模式增加更明显但不过火的渐变层次，提升品牌感和页面氛围
3. 继续收紧移动端 AI 助手页面，把输入区、顶部、卡片密度再压一轮

核心原则：

- 先补最容易回归的测试面，再扩测试数量
- 渐变效果优先轻量、克制、可统一，不做花哨皮肤系统
- 移动端 AI 助手继续以“少挤、少跳、少遮挡”为优先级

---

## 2. P1：测试补强

### 2.1 增加更多关键页面与组件测试文件 ✅
目标：

围绕最近版本高频变动区域，补齐缺失测试，降低后续迭代回归概率。

重点处理：

- 优先补 Assistant 页面核心交互测试
- 补 AI 信贷管家模式切换 / 首屏内容 / 预设问题测试
- 补还款管理页关键流程测试
- 补 WebDAV 恢复版本选择 / 版本化备份相关测试
- 对近期多次改动但覆盖薄弱的组件，至少补最小可运行测试文件

验收标准：

- 新增一批真实可执行测试文件，而不是只补空壳
- 核心高风险交互至少有基础覆盖
- `npm run test` 与 `npm run build` 可通过

---

## 3. P1：主题模式视觉增强

### 3.1 主题模式增加渐变效果
目标：

在现有主题模式基础上，为首页、卡片、关键容器补充统一的渐变层次，让界面更有气质，但保持金融工具应有的克制感。

重点处理：

- 为亮/暗主题分别设计轻量渐变策略
- 渐变优先作用于欢迎区、重点面板、状态条或大块容器
- 保持文字可读性与信息层级，不影响输入、图表、表格识别
- 避免局部拼贴感，尽量形成统一主题语言

验收标准：

- 页面整体观感更完整
- 渐变不影响可读性与操作性
- 明暗主题下都自然，不会廉价发光

---

## 4. P1：移动端 AI 助手进一步收紧

### 4.1 移动端 AI 助手页面继续收紧
目标：

在已有移动端减压基础上，再压一轮顶部、消息区、输入区和模式区密度，让小屏更顺手。

重点处理：

- 顶部模式切换与模型区进一步减少拥挤感
- 欢迎区 / 卡片区在小屏下继续减重
- 输入区、附件区、错误条、语义召回区继续压缩高度
- 关注键盘弹出后的可视区域与滚动体验

验收标准：

- 小屏下首屏更干净
- 输入与发问路径更顺手
- 不出现新的遮挡、换行爆炸或触控误触

## 6. 执行原则

1. 测试优先覆盖最近频繁改动的区域
2. 主题渐变先统一语言，再做局部精修
3. 移动端优化优先保证输入、阅读、滚动三件事
4. 每完成一项都更新计划状态并执行 `npm run build`
