# 0.5.9 - 工资工具独立入口调整计划

> 目标：将“工资计算工具”从当前“市场资讯”页面中拆分出来，提供独立入口，避免功能归类错位与认知混淆。

## 1. 范围

本轮只处理信息架构与页面挂载关系：导航、路由、页面承载结构与必要文案，不扩展新的工资计算功能。

---

## 3. 执行原则

1. 优先最小改动，不大改导航体系
2. 若能复用现有 `FinancePage`/资讯页结构，则优先复用
3. 页面命名要和内容一致，避免“点资讯进工具”的错位感
4. 每完成一项后同步更新计划状态

---

## 4. 验收标准

1. 工资工具拥有独立入口或独立承载页
2. 市场资讯页只保留资讯相关内容
3. 导航名称、页面标题、页面内容三者一致
4. 至少通过 `npm run build` 验收
