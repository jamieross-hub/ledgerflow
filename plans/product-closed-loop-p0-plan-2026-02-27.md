# LedgerFlow 产品闭环体验与 P0 验收标准方案

## 1. 核心价值一句话

帮你少花冤枉钱、看清现金流、并给可执行建议的财务助手。

对应三条用户结果：

- 记得更快
- 看得更清
- 决策更准

---

## 2. 闭环体验设计

闭环主链路：记录 -> 归类 -> 洞察 -> 建议 -> 执行动作 -> 结果反馈 -> 规则学习

```mermaid
flowchart LR
A[记录]
B[归类]
C[洞察]
D[建议]
E[执行动作]
F[结果反馈]
G[规则学习]

A --> B --> C --> D --> E --> F --> G --> B
```

### 2.1 各环节定义

1. 记录

- 目标：3 秒内完成常见场景记账
- 输入方式：手动快速录入、AI 文本、截图识别、账单导入
- 关键体验：默认值智能填充、最少字段必填、失败可撤销

2. 归类

- 目标：自动分类命中率持续提升
- 机制：初始规则 + 用户纠错回写 + 标签上下文补充
- 关键体验：用户只需纠错一次，后续同类消费自动修正

3. 洞察

- 目标：不堆数据，只给变化与风险
- 输出：本月异常、预算偏离、现金流压力窗口
- 关键体验：每条洞察附带原因和影响范围

4. 建议

- 目标：建议必须可执行
- 输出格式：建议动作 + 预期收益 + 执行成本
- 关键体验：同屏展示一键执行入口

5. 执行动作

- 目标：从看见问题到采取行动不超过 2 次点击
- 典型动作：
  - 一键创建本周餐饮封顶目标
  - 一键生成可取消订阅清单
  - 一键调低下月分类预算

6. 结果反馈

- 目标：用户知道动作是否有效
- 输出：执行后 3 天和 7 天变化对比
- 关键体验：明确告知节省金额和风险下降幅度

7. 规则学习

- 目标：系统越用越懂用户
- 回写来源：分类纠错、预算调整、忽略建议、完成动作
- 关键体验：可解释的学习结果与可撤销规则

## 3. 两条高价值闭环模板

### 3.1 外卖超预算闭环

1. 检测到餐饮分类超预算 20%
2. 输出建议：本周餐饮封顶金额
3. 用户一键创建目标
4. 每日提醒执行
5. 周末反馈节省金额
6. 若执行有效，提升类似建议优先级

### 3.2 订阅重复扣费闭环

1. 检测到同商户周期性自动扣费
2. 生成可取消清单
3. 用户标记已取消或保留
4. 下周期验证是否停扣
5. 回写订阅风险规则

## 4. P0 功能与验收标准

### P0-1 极速记账

验收标准

- 常见支出从进入页到保存成功 <= 3 秒
- 快捷入口覆盖首页、交易页、助手入口
- 保存失败有重试和回滚提示

度量

- 记账中位时长
- 首次记账完成率

### P0-2 自动分类与纠错学习

验收标准

- 自动分类命中率达到可用阈值并持续提升
- 用户纠错后同类样本后续自动应用
- 提供最近学习记录可查看与撤销

度量

- 自动分类命中率
- 纠错后二次命中率

### P0-3 月度预算与超支预警

验收标准

- 预算支持按分类设置与批量调整
- 超支前预警和超支后建议均可触达
- 预警至少包含分类、偏差比例、建议动作

度量

- 预算设置覆盖率
- 预警触达率
- 预警后执行率

### P0-4 账单导入稳定性

验收标准

- 微信、支付宝、银行账单导入成功率稳定
- 异常格式有可读错误提示
- 导入支持预览和重复数据处理

度量

- 导入成功率
- 导入失败原因分布

---

## 5. P0 实施顺序与依赖

1. 极速记账
2. 自动分类与纠错学习
3. 预算与预警
4. 导入稳定性

依赖关系

- 自动分类依赖高质量记录入口
- 预算预警依赖分类准确率
- 导入稳定性反向提升分类学习样本规模

## 7. P1 功能与验收标准

### P1-1 AI 周报与月报

验收标准

- 每周与每月自动生成一份结构化报告，覆盖收支变化、分类波动、异常事件与可执行建议
- 报告中每条建议都包含 建议动作 预期收益 执行门槛，且可一键进入对应执行入口
- 报告支持回看历史版本，用户可标记 建议已执行 暂不执行 与不相关，用于后续推荐学习

度量

- 周报与月报触达率
- 报告阅读完成率
- 报告建议执行率
- 执行后 7 天净支出改善率

### P1-2 异常提醒

验收标准

- 可识别至少三类高价值异常 大额突增 高频小额聚合 周期性重复扣费
- 每条异常提醒包含 触发原因 影响金额 影响时间窗 与建议动作
- 用户可对提醒执行 忽略 延后，系统在后续周期给出结果反馈并更新提醒策略

度量

- 异常识别准确率
- 异常提醒触达率
- 异常处理完成率
- 误报率与漏报率

### P1-3 30 天现金流预测

验收标准

- 输出未来 30 天现金流曲线并展示置信区间
- 对预测低于安全阈值的时间窗给出预警与至少两条可执行缓解动作
- 支持按账户与分类查看主要影响因子，解释预测变化来源

度量

- 30 天预测平均误差
- 低现金流预警命中率
- 预测建议执行率
- 预警后负现金流事件下降比例

## 8. P2 功能与验收标准

### P2-1 家庭与伴侣协作账本

验收标准

- 支持多成员共享账本并提供成员级权限 查看 录入 审核 管理
- 协作场景下支持成员责任分配与提醒，减少重复记账与漏记
- 所有协作操作有可追溯记录，关键动作支持撤销与冲突解决

度量

- 协作账本创建率
- 成员周活跃率
- 协作场景重复记账率
- 协作冲突解决时效达成率

### P2-2 目标储蓄挑战 游戏化

验收标准

- 支持按目标金额与周期创建储蓄挑战，并拆解为周任务与月里程碑
- 挑战过程提供进度反馈与偏离预警，支持一键应用预算与消费限制动作
- 达成与中断都形成复盘卡片，沉淀可复用策略模板

度量

- 挑战创建率
- 挑战阶段留存率
- 挑战达成率
- 挑战期间非必要支出下降比例

### P2-3 模板市场

验收标准

- 提供预算模板 记账规则模板 挑战模板 的浏览预览应用流程
- 模板应用后可自动映射到账户分类标签结构，允许二次编辑
- 模板效果可被量化追踪，支持按执行效果排序与推荐

度量

- 模板浏览到应用转化率
- 模板 30 天留存使用率
- 模板驱动建议执行率
- 高评分模板占比

## 9. 分阶段验收闸门

P1 进入实现前闸门

- P1 三项功能均具备可度量口径并已定义数据采集点
- 关键动作入口已映射到现有页面或明确新增页面
- 异常提醒与预测能力的误报容忍策略已定义

P2 进入实现前闸门

- 协作权限模型与审计追踪字段已完成领域建模
- 游戏化挑战与模板市场的数据结构可支持版本演进
- P2 指标已与 P0 P1 指标体系打通，可做跨阶段归因分析
