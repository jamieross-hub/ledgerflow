# LedgerFlow 优化计划（v0.4.3 UI 优先版）

> 更新日期：2026-03-06  
> 状态：当前阶段以 **UI 美观度、统一性、专业感** 为第一优先级。

---

## 1. 当前优化方向

LedgerFlow 下一阶段不再优先推进深功能扩展，先集中处理：

1. **整体 UI 美观度提升**
2. **页面视觉一致性统一**
3. **图表 / 卡片 / 按钮 / 状态条的专业感提升**
4. **文本溢出、间距拥挤、局部毛刺的系统性清理**

功能型优化（如导入链路三段式、Transactions 聚焦流水、服务端安全闭环）保留，但顺序后移。

---

## 2. 当前策略

### 核心原则

先把产品做得 **顺眼、统一、稳重、像正式产品**，再继续扩展复杂功能。

当前 UI 优化重点不是“单点修 bug”，而是：

- 统一视觉密度
- 统一卡片层级
- 统一图表表现形式
- 统一按钮与筛选器风格
- 统一状态反馈组件样式
- 清理所有影响专业感的视觉噪音

## 4. 当前最高优先级（P0）

### 4.1 Dashboard 视觉统一化
目标：

把 Dashboard 从“功能堆叠”整理成“视觉统一、信息分层清楚、适合持续浏览”的正式面板。

重点处理：

- 收支趋势（决策视图）表现形式统一
- 净资产曲线（含每月 Δ）卡片间距与层级统一
- 图表说明、标签、趋势文字不再拥挤
- 信息卡片之间的留白统一
- 异常提醒 / 洞察 / 趋势模块的卡片语言统一

验收标准：

- 无明显文字溢出
- 卡片之间不粘连、不拥挤
- 图表标签不抢主视觉
- 页面整体观感更专业、更克制

---

### 4.2 Smart Budget 页面美观度整理
目标：

把 Smart Budget 从“能用”提升到“整洁、稳定、有专业感”。

重点处理：

- 筛选器、状态标签、卡片间距统一
- 柱状图、趋势图、状态区块风格统一
- 预算进度卡的密度与层级统一
- 说明文案与按钮排列优化
- 小屏和窄屏下换行、遮挡、拥挤问题修复

验收标准：

- 无明显方块残影、溢出、遮挡
- 卡片和图表布局一致
- 状态标签 / 颜色表达更稳定
- 页面看起来不像多个临时 patch 拼出来的

---

### 4.4 还款管理严谨化改造 ✅
目标：

把还款管理从“提醒型页面”升级为“严谨可追溯的贷款/负债管理台账”。

重点覆盖：

- 还款日期监控（账单日 / 还款日 / 提前提醒 / 逾期识别）
- 还款金额监控（最低还款 / 应还总额 / 本金 / 利息 / 手续费）
- 计算公式透明化（等额本息 / 等额本金 / 分期 / 自定义）
- 还款记录方式（手动登记 / 自动识别 / 部分还款 / 提前还款 / 撤销）
- 扣款账户跟踪（从哪个账户扣、扣后余额联动、账户不足预警）
- 还款后闭环（剩余本金、剩余期数、下期计划、历史记录可追溯）

第一步执行内容：

1. 梳理当前还款管理页面已有字段与缺口
2. 明确“贷款主体 / 计划 / 记录 / 校验”四层结构
3. 输出字段补全建议与页面结构调整建议

验收标准：

- 能明确列出当前还款管理的数据缺口
- 能落出严谨版字段模型草案
- 后续实现时不再依赖模糊描述

### 4.5 账单详情附件上传（依赖 WebDAV） ✅
目标：

在账单详情中支持插入附件，并将附件统一上传到 WebDAV，用于保留票据、截图、合同、对账单等原始凭证。

重点约束：

- 仅在 **WebDAV 已正确配置** 时允许上传附件
- 未配置 WebDAV 时，账单详情中附件上传入口必须禁用或明确提示不可用
- 附件上传成功后，账单详情可查看附件名称、时间与访问入口
- 附件与账单记录需要建立稳定关联，避免孤立文件

执行要求：

1. 账单详情页增加“插入附件 / 上传附件”入口
2. 上传目标统一走 WebDAV
3. 未配置 WebDAV 时禁止上传，并提示先去数据库/WebDAV 设置完成配置
4. 需考虑附件元信息：文件名、上传时间、关联账单 ID、远程路径
5. 后续可扩展为多附件，但第一阶段至少支持单附件稳定上传

验收标准：

- WebDAV 未配置时，无法上传附件，且提示清晰
- WebDAV 已配置时，可成功上传并在账单详情中展示附件信息
- 附件与账单详情的关联关系清晰，不出现“文件上传成功但页面无记录”的断链

## 5. 第二优先级（P1）

### 5.1 全站视觉规范收敛
目标：

把近期散落的 UI 修复汇总成一套统一风格，而不是继续零散修补。

包括：

- 卡片圆角体系统一
- 按钮尺寸与 padding 统一
- 图表标题/副标题/图例风格统一
- 状态标签颜色逻辑统一
- 小字号说明文案的字体与颜色统一
- 常见列表行高、卡片间距、空状态风格统一

说明：

这一步是为了降低后面继续出现“某一块看起来不协调”的概率。

---

### 5.2 响应式与窄屏观感治理
目标：

避免桌面端看着还行，窄屏下又开始挤、乱、溢出。

重点：

- dashboard 图表区
- smart budget 卡片区
- assistant 输入与状态区
- 交易页筛选工具区

## 8. 执行原则

1. **优先解决用户一眼就能感知的不协调问题**  
2. **优先统一体验，不堆功能入口**  
3. **每次改动都必须 build 验证后再提交**  
4. **每轮尽量形成可单独推送的小提交**

---

## 9. 当前结论

LedgerFlow 当前阶段的主方向已经调整为：

> **先做 UI 美观度与一致性，再继续推进深功能优化。**

当前最应该优先投入的，不是继续堆新功能，而是把：

- dashboard
- smart budget
- assistant

这三块做得更统一、更专业、更有成品感。
