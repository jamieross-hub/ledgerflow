# LedgerFlow v0.4.3 发布说明

发布日期：2026-03-06

## 版本定位

`v0.4.3` 是一个 **UI 优先版**。

这一版不追求继续堆功能，而是集中把 LedgerFlow 做得更统一、更稳重、更像一个可持续使用的正式产品：

- 统一页面视觉语言
- 优化高频页面的层级与密度
- 收紧手机端与窄屏体验
- 补齐交易流与导入流的可用性闭环
- 保留后续增强项，但不把它们混进当前发版阻塞范围

## 亮点更新

### 🎨 Dashboard / Smart Budget / 全站视觉统一

- 完成 Dashboard 视觉统一化，整理图表、说明文案、卡片留白与信息层级
- 完成 Smart Budget 页面美观度整理，统一卡片、状态标签、趋势区块与筛选区观感
- 完成全站视觉规范收敛：
  - 统一 `panel / card` 基线
  - 统一小字号说明文案风格
  - 统一状态 pill / badge 语言
  - 收敛常见面板阴影、圆角、边框和间距表现

### 📱 响应式与手机端观感治理

- 补强 Dashboard、Smart Budget、Assistant、Transactions 的窄屏适配
- 还款管理页面新增手机端折叠分段结构，降低长表单直铺到底的压迫感
- 优化交易页筛选区和 Assistant 顶部/状态区在小屏下的排布与点击体验

### 💳 还款管理更严谨

- 还款结果支持区分：`partial / normal / overpayment`
- 成功/警告提示更准确，避免“还了但说不清还得怎么样”的体验
- 最近还款记录补充结果标签与台账关联，追溯性更完整

### 📎 账单详情支持附件上传（依赖 WebDAV）

- 交易详情中新增“插入附件 / 上传附件”入口
- 附件统一上传到 WebDAV
- 未配置 WebDAV 时会明确提示不可上传
- 上传成功后可在账单详情查看附件名称、上传时间与远程路径

### 📒 Transactions 页更聚焦流水本身

- 交易页默认聚焦流水列表
- 右侧图表/洞察首次进入默认关闭，避免首屏注意力被分散
- 保留按需展开入口，不牺牲已有能力

### 📥 导入链路整理为三段式

导入体验明确拆分为三步：

1. 选择来源与模式
2. 预检确认影响范围
3. 正式写入并生成导入报告

本次同步补齐了入口文案、预检确认语义和完成反馈，让导入链路更像一个完整流程，而不是散落的几个按钮。

## 技术说明

- 当前前端版本号已更新为 `0.4.3`
- 本次改动均已完成构建验证（`npm run build`）
- 构建仍存在 Vite chunk size warning，但未阻断发版；属于后续性能优化项，不影响本次功能与 UI 交付

## 本版未纳入阻塞项

以下事项不作为 `v0.4.3` 的发版阻塞条件：

- **6.3 语义召回可观测性第二阶段**：属于后续增强项
- **6.4 安全收口（服务端）**：当前仓库不适用，不纳入 LedgerFlow `v0.4.3` 发版判断

## 致谢

这一版主要是一次“把产品收紧”的工作：
不是加最多功能，而是把已经存在的能力做得更顺、更稳、更像成品。

如果 `v0.4.2` 更像能力铺底，`v0.4.3` 就更像一次真正面向发布的整理版。
